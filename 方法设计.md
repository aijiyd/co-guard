**背景假设**： 在多智能体系统中，攻击者将恶意目标拆解为多个子任务并分发给不同 Agent，试图通过整合各 Agent 的输出来完成攻击。

### 方法设计

##### 模块一：语义解析器，负责进行实体抽取和标准化

**输入**：用户查询

**工作流程**：

阶段1：使用小样本提示LLM+prompt 从自然语言中抽取实体和关系三元组

阶段2：prompt + LLM 为提取出的关系生成一个自然语言定义

阶段3：向量化关系定义和已有的 Schema，计算定义向量与目标Schema中关系向量的相似度，检索Top-k 个候选关系，在提取出的关系集合内部进行聚类

**输出**：标准化后的三元组

##### 模块二：知识图谱构建

使用 Neo4j 图数据库，每出现一个用户查询就更新知识图谱

##### 模块三：推理层

根据新加入的用户查询为中心，进行上下文子图提取，提取有关联的节点和关系

使用 LLM 将图转换为自然语言描述

使用 LLM 判断是否恶意

是则拒绝回答
