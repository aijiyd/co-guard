# 1 绪论

## 1.1 文献综述

### 1.1.1 大语言模型智能体安全研究现状

大语言模型的应用形态正在从单次问答扩展为能够调用工具、保存记忆、执行规划并与环境交互的智能体系统。Yu 等（2025）在可信 LLM Agent 综述中将基于大语言模型的智能体划分为内部模块与外部交互模块，指出单一 LLM、单智能体和多智能体系统在攻击面、可信度评估和防御机制上存在层级差异[1]。该研究表明，智能体的安全问题不再只由模型本身决定，还受到记忆、工具、用户、其他 Agent 和外部环境的共同影响。与普通聊天机器人相比，智能体具有更长的任务链和更复杂的信息流，这使得攻击可以沿着组件交互过程传播。

针对智能体系统的对抗鲁棒性，Wu 等（2024）提出 Agent Robustness Evaluation 框架，将智能体表示为中间输出在组件间流动的图结构，并在网页智能体环境中验证了文本注入、图像扰动和组件级传播攻击的影响[2]。该工作说明，反思机制、树搜索、评估器等推理时组件虽然可以提升良性任务性能，但也可能扩大攻击入口，使对抗信息在系统内部被放大。Baker 等（2025）从推理模型监控角度讨论了通过观察思维链检测不当行为的可能性，同时指出过度优化监控指标可能导致模型隐藏真实意图[3]。上述研究共同说明，面向智能体的安全防御需要关注系统级过程，而不是只检查最终输出。

多智能体场景中的安全问题也逐渐受到关注。Chern 等（2024）研究了多智能体辩论对对抗提示的缓解效果，实验显示当被越狱或能力较弱的模型与安全对齐模型进行辩论时，生成内容的毒性可能下降，但这种机制对不同攻击主题和模型组合的效果并不稳定[4]。Song 等（2024）提出 Audit-LLM，将多智能体协作用于日志型内部威胁检测，通过任务分解、工具构建和证据辩论缓解长日志和解释可信度问题[5]。此类研究表明，多智能体协同既可能成为攻击者利用的复杂系统，也可能成为防御者提升审查能力的工程手段。

### 1.1.2 越狱攻击与子任务拆解攻击研究现状

越狱攻击研究最初主要围绕对抗提示、角色扮演、后缀优化和提示注入展开，其目标是诱导安全对齐模型输出本应拒绝的内容。Li 等（2024）提出 DrAttack，发现将恶意提示拆分为多个子提示并通过良性示例进行隐式重构，可以降低模型对恶意词组组合的注意，从而提高越狱成功率[6]。该方法包含提示分解、上下文学习重构和同义词搜索等环节，说明有害意图并不一定以完整句子的形式出现，而可以隐藏在碎片化语义成分中。DrAttack 的实验进一步表明，该方法在多种开源和闭源模型上都取得了高于既有方法的攻击成功率，并且在更少查询次数下仍保持较强攻击效果，其中对 GPT-4 的攻击成功率达到 80%[6]。这一结论正对应本文试图解决的问题，即防御系统不能只检查单轮 query 的表面风险，还需要识别多个低风险片段在历史上下文中是否正在重新组装为完整攻击目标。

Srivastav 和 Zhang（2025）进一步将拆解思想扩展到 Agent 驱动的多轮攻击场景，提出 Safe in Isolation, Dangerous Together，说明攻击者可以借助问题拆解器、子问题回答器和答案组合器，将有害目标拆成多个看似无害的子任务。该研究报告了多模型上的高攻击成功率，并指出现有安全架构缺少整体上下文感知能力[7]。对本文而言，这一结论具有直接启发意义：若防御系统只判断每一轮输入是否违规，就可能允许所有局部子任务通过；若系统能够识别多个子任务之间的阶段依赖和主题延续，就有机会在攻击链闭合前触发拒绝。

多模态和复合智能体攻击也说明，攻击的表现形式正在从文本级越狱转向系统级过程控制。Oh 等（2025）提出 UniGuard，针对多模态越狱攻击设计通用安全护栏，证明图像与文本中的有害信号需要联合考虑[8]。Wu 等（2024）的智能体鲁棒性研究则表明，对抗信息可以借助网页文本、图像和中间组件输出在系统内流动[2]。虽然本文聚焦文本多轮子任务，但这些研究说明，未来防御系统需要保留跨组件、跨轮次甚至跨模态的上下文线索。

国内综述研究也在快速跟进这一方向。黄河燕等（2025）从安全分类、评估、归因和缓解四个层面系统梳理了大语言模型安全问题，姜毅等（2025）进一步从完整生命周期总结了安全与隐私风险，梁思源等（2024）则围绕越狱攻击与防御方法及评测基准做了综合评述[44-46]。这些工作说明，国内研究已开始从单点攻击现象转向系统化风险框架，但针对多轮拆解攻击的跨轮结构化检测仍缺少专门设计。

### 1.1.3 大语言模型安全防御研究现状

现有大语言模型防御主要包括模型对齐、提示词优化、输入输出过滤、安全护栏和多智能体审查等方向。Li、Wang 和 Zhou（2024）提出 Robust Prompt Optimization，通过优化可迁移的防御后缀降低越狱攻击成功率[9]。这类方法把防御逻辑注入系统提示或输入处理阶段，具有部署成本低的优势，但主要面向当前输入和当前模型响应，对历史多轮上下文的结构化分析有限。

外部安全护栏是部署型防御中的重要路径。Zeng 等（2024）提出 AutoDefense，使用多个小型开源语言模型 Agent 分工协作，对大模型响应进行安全过滤，并在越狱攻击场景中显著降低攻击成功率[10]。Oh 等（2025）的 UniGuard 则强调护栏机制应具备通用性、跨模型适配性和较低推理开销[8]。Yu 等（2025）在综述中也将单模型过滤、多智能体盾牌和安全评估作为 Agent 防御的重要分支[1]。此类工作说明，外部护栏适合在不重新训练目标模型的情况下增强安全性。

现有护栏方法仍存在上下文视野不足的问题。单模型过滤器通常基于当前输入或当前回答判断风险，多智能体审查虽然引入了更多判断角色，但并不必然保留跨轮任务链。对于子任务拆解攻击，当前轮次可能只是询问材料、术语、背景或一般步骤，本身并不具有明确违规特征。若安全系统无法把当前轮次与历史轮次中的目标识别、能力获取、规避控制和执行准备联系起来，就难以判断其在完整攻击链中的作用。本文选择以图谱形式保存多轮上下文，正是为了弥补单轮护栏和普通多 Agent 审查对结构化意图链建模不足的问题。

与国外偏重提示攻防和越狱基准的研究相比，国内在知识图谱构建与推理方面已形成较成熟的综述积累。漆桂林等（2017）、刘峤等（2016）和马忠贵等（2020）系统总结了知识抽取、知识融合、知识推理与质量挑战，封皓君等（2021）进一步对面向知识图谱的推理方法进行了分类评述[47-50]。这些研究为本文采用图结构保存多轮上下文、并通过路径推理识别跨轮恶意闭合提供了方法论基础。

### 1.1.4 知识图谱构建与动态上下文建模研究现状

知识图谱以实体和关系为基本单元，能够把自然语言或日志中的对象、动作和依赖关系组织为可检索、可推理的结构。Pan 等（2024）系统总结了大语言模型与知识图谱结合的路线，指出知识图谱可以为大语言模型提供外部知识、事实约束和可解释推理路径，大语言模型也可以反过来辅助知识图谱构建、补全和问答[11]。AGENTiGraph（Zhao 等，2025）进一步展示了多智能体系统如何通过自然语言进行知识图谱查询、更新和管理，说明 LLM 与 KG 的结合适合多轮交互和动态知识集成场景[12]。

自然语言到知识图谱的转换依赖实体识别、关系抽取和关系规范化。Kumar 等（2020）基于预训练语言模型学习实体感知关系，并将文本中的实体关系转化为三元组后写入图数据库[13]。Zhang 和 Soh（2024）提出 Extract, Define, Canonicalize 框架，将知识图谱构建拆分为开放式信息抽取、关系定义生成和后置规范化三个阶段，缓解了大 schema 难以完整放入提示词和开放域文本关系表达不统一的问题[14]。该方法与本文的语义解析模块高度相关，因为子任务拆解攻击中的风险关系常以“查询”“了解”“避免”“准备”“获取”等多种自然语言形式出现，直接依赖固定关键词容易造成漏检。

系统安全领域的溯源图研究也为本文提供了结构化建模依据。Milajerdi 等（2019）提出 HOLMES，通过关联系统审计日志中的可疑信息流，实时构建高层攻击场景图，用于检测高级持续性威胁[15]。HOLMES 的核心思想是：单个底层事件未必足以判定攻击，但多个事件在信息流和因果关系上连接后，可以呈现更完整的攻击阶段。多轮子任务拆解攻击与这一问题具有相似结构。攻击者输入的单个子任务可能只是局部、低风险的信息请求，但多个子任务在时间顺序和语义依赖上组合后，可能共同指向完整有害目标。因此，将多轮查询写入上下文图谱，并围绕当前查询检索相关子图，是一种具有工程可行性的防御路线。

### 1.1.5 图推理与可解释安全判定研究现状

大语言模型在复杂推理中容易受到幻觉、知识缺失和上下文噪声影响，知识图谱可以通过结构化路径为其提供外部约束。Sun 等（2024）提出 Think-on-Graph，将大语言模型视为在知识图谱上交互式探索实体和关系的智能体，通过图搜索发现可能的推理路径，再基于检索到的知识作答[16]。Luo 等（2024）提出 Reasoning on Graphs，通过“规划、检索、推理”框架先生成关系路径规划，再沿知识图谱检索具体推理路径，使最终推理更忠实于图结构证据[17]。He 等（2024）的 G-Retriever 也说明，面向文本图的检索增强生成可以改善图理解与问答任务[18]。

这些图推理研究为本文提供了两方面启发。其一，图谱中的关系路径可以作为大语言模型判断的证据，而不是把长上下文直接拼接给模型。其二，关系类型本身具有推理价值。对于多轮拆解攻击，风险往往不只体现为某个实体名称，而体现为目标识别、漏洞探索、能力获取、规避控制、执行准备和结果实现等关系阶段的组合。本文因此采用 relation-first 图搜索策略，让系统优先扩展风险关系和跨轮桥接关系，而不是只依据实体相似度检索邻居。

综合现有文献可以看出，国内外研究已经在智能体安全、越狱攻击、防御护栏、知识图谱构建和图推理方面取得了较多成果，但这些方向之间仍存在连接不足。越狱攻击研究揭示了子任务拆解的有效性，却较少给出面向多轮会话的结构化防御流程。安全护栏研究降低了部分越狱风险，但多以单轮输入输出为审查对象。知识图谱和图推理研究提供了可解释路径证据，但更多服务于问答、知识管理和通用推理任务。系统安全溯源研究证明了图结构关联碎片化行为的价值，但其输入通常是系统审计日志而非自然语言子任务。本文在上述研究基础上，将 EDC 风格语义抽取、会话级上下文图谱、relation-first 多跳搜索和上下文组装评分器结合起来，用于识别多轮子任务共同组装出的潜在恶意意图链。

## 1.2 研究框架

本研究定位为面向多轮拆解攻击检测的探索性研究。研究目标不是证明当前实现已经形成可直接部署的成熟防御方案，而是验证上下文知识图谱与图推理能否为跨轮风险判定提供可追溯证据，并识别该路线在长期共享上下文下的主要局限。

围绕这一定位，本文将研究目标具体化为三个方面。其一，构建能够保留多轮语义关联的上下文知识图谱，使拆解后的局部子任务不再被孤立处理。其二，设计面向跨轮风险链的图推理机制，判断当前查询是否与历史查询形成阶段推进或恶意闭合。其三，在单会话、跨会话和长期共享上下文三类场景下评估该路线的召回能力、截断时机与误拒边界，为后续改进提供可复核的实验依据。

技术路线以自然语言查询为起点，通过结构化抽取、图谱持久化、上下文检索与逻辑裁决形成闭环。第一部分是语义解析与标准化模块。系统从当前输入中提取实体、关系和三元组，并借鉴 EDC（Extract, Define, Canonicalize）思想生成关系定义，把开放式关系逐步映射到内部 Schema。第二部分是上下文图谱构建模块。系统将标准化后的三元组、查询节点和实体节点写入图结构，并区分 `session_id` 与 `context_id` 两类上下文标识，使单会话隔离和跨会话共享可以在同一框架中表达。第三部分是图推理与安全判定模块。系统围绕当前查询检索局部子图，在最近历史窗口内分析主题锚点、任务阶段和上下文对齐关系，再结合 relation-first 多跳搜索、规则约束与大模型复核，尝试判断当前查询是否推进或闭合多阶段风险链。

研究内容包括三个层面。其一，构建面向多轮查询的语义抽取与关系规范化流程，减少自然语言表达差异对图推理的干扰。其二，设计支持 `session_id/context_id` 双层边界的上下文知识图谱，刻画多轮交互中的查询、实体和关系累积过程。其三，设计 relation-first 搜索与上下文组装评分机制，考察系统在不同共享上下文条件下的召回能力与误拒边界。

在实验评估环节，本文构建了三个复杂度递进的对抗测试场景。场景一作为单链路顺序注入基线，将每条攻击链绑定于独立 `session_id`；场景二在单链路基础上加入跨会话轮换和良性任务噪声；场景三进一步模拟单一长期用户上下文、多条隐藏攻击链交替推进及共享噪声并存的环境。三类场景既用于考察该路线对跨轮风险链的识别能力，也用于暴露其在长期共享上下文下的误拒问题。系统在运行期间仅可获取查询语句 `query`、会话标识 `session_id` 与共享上下文标识 `context_id`，无法访问评测器离线维护的隐藏目标标识 `hidden_goal_id`，以保持评估边界与真实运行条件一致。

## 1.3 术语说明

### 1.3.1 大语言模型

大语言模型是指基于大规模语料训练、能够执行自然语言理解与生成任务的深度学习模型。本文讨论的大语言模型不仅包括普通对话模型，也包括被集成到智能体系统中、能够参与任务分解、工具调用和多轮交互的模型组件。本文关注的安全问题不只来自模型单次回答，而是来自模型在连续交互中被诱导输出可组合的危险信息。

### 1.3.2 智能体与多智能体系统

智能体是指能够接收任务、理解环境、规划步骤并执行操作的软件实体。在大语言模型应用中，智能体通常由语言模型、提示词、记忆模块、工具接口和执行逻辑组成。多智能体系统由多个智能体协同完成复杂任务，不同智能体可能承担规划、检索、执行、审查等角色。本文中的多智能体场景强调任务在多个角色之间被拆分和流转后产生的安全风险。

### 1.3.3 子任务拆解攻击

子任务拆解攻击是指攻击者将一个原本明显有害的目标拆分为多个表面中立或低风险的子问题，并通过多轮输入或多智能体分发逐步获取完成攻击所需的信息。每个子任务单独观察时可能表现为概念解释、材料查询、步骤讨论或风险分析，但多个子任务组合后可能还原出完整恶意意图。本文的防御目标正是识别这种跨轮组合风险。

### 1.3.4 知识图谱

知识图谱是一种以节点和边表示知识的结构化数据形式。节点通常表示实体、事件或查询，边表示节点之间的语义关系、依赖关系或时间关系。本文中的上下文知识图谱以用户查询节点、实体节点和关系边为基本结构，用于保存多轮会话中的语义上下文，并为后续图推理提供证据基础。

### 1.3.5 语义三元组

语义三元组是知识图谱构建中的基本表示形式，通常写作 subject-relation-object，即主语、关系和宾语。对于用户输入“攻击者绕过安全策略获取 API 密钥”，系统可能抽取出“攻击者 - 绕过 - 安全策略”和“攻击者 - 获取 - API 密钥”等三元组。本文将原始三元组进一步规范化，使不同文本表达能够映射到统一的风险关系类型。

### 1.3.6 EDC

EDC 是 Extract, Define, Canonicalize 的缩写，表示抽取、定义和规范化三个阶段。Extract 阶段从文本中抽取开放式实体关系三元组，Define 阶段为关系生成自然语言定义，Canonicalize 阶段将开放式关系映射到目标 schema。本文借鉴该思想构建语义解析模块，用于处理多轮查询中的关系表达差异。

### 1.3.7 relation-first 图推理

relation-first 图推理是本文采用的图搜索策略。传统图搜索往往从实体相似性出发扩展邻居，而本文任务更关注关系是否推动了风险链。因此，系统在搜索上下文子图时优先关注风险关系、阶段关系和跨轮桥接关系，例如目标识别、能力获取、规避控制、执行准备和敏感资源获取。该策略能够让系统更快定位与拆解攻击相关的路径证据。

### 1.3.8 上下文组装评分器

上下文组装评分器是本文的上下文组装评分机制。它不要求系统恢复真实攻击链编号，而是判断当前查询写入可见上下文后，是否与近期历史查询在危险主题、内容锚点和任务阶段上形成闭合风险链。在单链基线场景中，组装范围通常等同于单条攻击链的独立上下文；在长期共享上下文场景中，系统则以共享 `context_id` 下的最近历史窗口为主要观察范围，并结合主题对齐和 anchor 对齐尝试缓解跨链污染。第四章结果表明，这一机制尚不能避免长期共享上下文下的高误拒。该机制用于弥补单条路径搜索对长链路攻击识别不足的问题。

### 1.3.9 防御成功率与攻击穿透率

防御成功率表示系统在多轮子任务输入过程中成功触发拒绝的攻击样本比例。攻击穿透率表示所有子任务输入完毕后系统仍未拒绝的样本比例。本文采用顺序注入方式评估系统效果，即对每条样本逐轮输入拆解后的子任务，只要任意一轮触发拒绝，就判定该样本防御成功；若全部轮次均被允许，则判定攻击穿透。
