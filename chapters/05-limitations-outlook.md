# 5 现有不足和未来展望

## 5.1 现有不足

当前系统的第一类不足来自语义抽取层。场景一和场景三中的部分 bypass 仍集中表现为危险主题丢失、关系表达被弱化或关键对象未被稳定保留。这说明 EDC 风格抽取与 refinement 虽然已经能够覆盖大量拆解攻击子任务，但面对弱显式表达、跨句省略和高伪装查询时，模块一仍可能把后续图推理所需的核心线索抽得过浅。只要主题和关系在入口阶段发生信息损失，后续图谱写入和图搜索就难以恢复完整风险链。

第二类不足来自共享长期上下文下的误拒绝问题。第四章结果表明，Co-Guard 在场景三和全量实验中已经具备较高攻击链召回能力，但良性误拒率仍然偏高。其根本原因在于，同一长期上下文中会并存多条隐藏攻击链、良性任务噪声和大量历史残留信息。即使系统已经引入 recent-history 窗口、主题对齐、anchor 对齐和 adequacy 修正，当前 query 仍可能受到历史风险痕迹的牵连而被过早判定为恶意。这说明系统虽然已经能够识别跨轮组装攻击，但对长期上下文污染的抑制仍不充分。

第三类不足体现在模型迁移能力与实验覆盖范围上。当前正式结果表明，`Qwen2.5-7B-Instruct` 在本文任务上明显优于 `Llama-3.1-8B-Instruct`，而部分候选模型在统一 `system + user` 调用协议下兼容性不足。这意味着 Co-Guard 的整体表现仍然受到基座模型语义理解能力和接口兼容性的显著影响。与此同时，当前实验尚未对 AutoDefense、UniGuard、Robust Prompt Optimization 等外部方法做统一边界下的复现比较，拆解语料本身也未进行单独的人类一致性标注，因此第四章结果更适合作为内部路线验证，而不是对外部方法优劣的充分证明。现有 benign 噪声主要用于模拟一般背景任务，尚未系统引入同主题 benign 子任务链这一更强干扰条件，因此当前结果对误拒绝边界的刻画仍然偏保守。

## 5.2 未来展望

后续工作应继续提升语义抽取层对弱显式攻击线索的保留能力。一方面，可以在当前 EDC 流程上进一步引入更细粒度的实体边界控制和关系不确定性传播机制，使“主题已出现但动作尚不充分”“动作已出现但对象仍模糊”这类中间状态也能以结构化方式进入图谱。另一方面，可以围绕高风险关系与中性分析关系之间的边界，细化规范化策略，减少 benign 查询在抽取阶段被压成过强攻击关系的概率。

共享长期上下文下的污染抑制仍是最核心的优化方向。未来可在现有 `session_id/context_id` 双层边界基础上，进一步探索按主题分桶的局部记忆、基于链路置信度的历史衰减和更精细的上下文拆分策略，使系统不仅能判断“当前 query 是否与历史对齐”，还能够更稳定地区分“与哪一类历史对齐”。若这一问题得到改善，Co-Guard 才有可能在保持高召回的同时，把良性误拒率进一步压低到更可用的范围。

评测框架本身也仍有扩展空间。后续实验应优先补入统一输入输出边界下的外部基线复现，并为拆解语料增加人工抽查或一致性标注，以提高实验结论的外部可比性。与此同时，可在现有三类场景之上补入同主题 benign 查询、同主题 benign 子任务链和更长周期的用户任务混排，用更严格的背景噪声测试系统的误拒绝边界。模型层面则应继续比较更多兼容基座模型，并统一记录运行时元数据、延迟与资源开销，使后续研究不仅能比较检测效果，也能比较部署成本。沿着这些方向推进，Co-Guard 才能从“验证方法可行性”进一步走向“面向真实长期交互环境的可部署防御系统”。
