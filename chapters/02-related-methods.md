# 2 相关理论与方法

## 2.1 大语言模型安全防御

大语言模型安全防御的目标，是在模型理解和生成能力不断增强的条件下，降低其被诱导生成违法、有害或违背系统策略内容的概率。传统问答模型中的安全问题多表现为单轮输入触发单轮输出，防御系统可以围绕当前提示词、当前响应或二者的组合进行检测。智能体系统改变了这一假设。模型在智能体中往往承担规划、调用工具、检索记忆和协调多个组件的角色，安全风险可能出现在任务分解、信息检索、工具调用和结果汇总的任一阶段。Yu 等（2025）将 LLM Agent 的可信度扩展到内部模块和外部交互模块，说明安全检测对象已经从单一模型转向由模型、记忆、工具、用户和环境组成的复合系统[1]。

模型对齐是大语言模型安全防御的基础路径。该类方法通过监督微调、人类反馈强化学习、安全数据增强或拒绝样本训练，使模型在面对有害请求时学习拒绝边界。对齐方法能够从模型内部改善安全行为，但其更新成本较高，对已经部署的应用系统不够灵活。越狱攻击研究表明，攻击者可以通过提示重写、角色扮演、后缀优化或语义拆分降低有害意图的可见性，使模型绕过训练阶段形成的拒绝模式。Li 等（2024）的 DrAttack 说明，将恶意提示拆分成多个语义碎片后，模型对恶意词组组合的注意会下降，这暴露出仅依赖模型内部对齐的局限[6]。

提示层防御试图在不修改模型参数的情况下增强模型拒绝能力。Robust Prompt Optimization 通过优化防御后缀，使模型在面对自适应越狱攻击时更倾向于安全拒答[9]。UniGuard 则在多模态场景中通过文本后缀和图像扰动构造通用安全护栏，降低模型生成有害内容的概率[8]。这类方法部署成本较低，适合以系统提示、输入改写或附加安全约束的方式接入模型。然而，提示层防御主要作用于当前输入，其效果依赖模型对当前上下文的理解能力。当攻击者将危险目标拆散到多个轮次时，当前输入可能并不直接包含完整风险，提示层防御难以凭单条文本判断其在整个会话中的作用。

外部护栏系统是一种更适合工程部署的防御方式。它通常位于用户输入、模型输出或工具调用前后，通过分类器、规则、独立审查模型或多智能体协作机制判断是否放行。AutoDefense 使用多个小型语言模型 Agent 分工过滤有害响应，在越狱攻击场景下能够降低攻击成功率[10]。多智能体辩论方法则让多个模型对同一问题进行讨论和互相修正，在部分对抗提示上可以降低有害输出概率[4]。这些方法证明了外部审查对模型安全的价值，但其基本判断单元仍多为当前请求或当前响应。对于多轮拆解攻击，防御系统需要额外保存历史查询之间的主题延续、阶段推进和语义依赖。

国内相关综述也强调了这一趋势。黄河燕等（2025）将大语言模型安全问题划分为模型自身安全和生成内容安全两类，并系统讨论了评估、归因和缓解路径[44]。姜毅等（2025）进一步把安全与隐私风险放到预训练、微调和部署全流程中考察[45]。梁思源等（2024）则指出，越狱攻击与防御研究已经从单一提示技巧扩展到生命周期防御与评测基准建设[46]。这些结论与本文的切入点一致，即防御对象需要从单轮文本转向跨轮过程。

本文采用的 Co-Guard 属于外部护栏系统，但检测对象不是孤立输入，而是当前查询与历史上下文共同形成的结构化意图链。系统并不修改目标大语言模型权重，而是在用户查询进入后执行语义抽取、图谱写入和图推理判定。若当前查询与历史查询在图中形成风险路径，系统在模型生成前触发拒绝。该思路把防御边界从“单轮文本是否违规”扩展为“多轮会话是否正在组装潜在恶意目标”，更适合本文关注的子任务拆解攻击。

## 2.2 多轮子任务拆解攻击

多轮子任务拆解攻击是指攻击者将原本可以被安全系统直接识别的有害目标，拆分为若干个表面中立或低风险的子任务，并通过多轮交互逐步获取完成目标所需的信息。该类攻击利用了安全检测中的局部视野问题。单个子任务可能表现为概念解释、材料查询、原理说明、风险讨论或防护建议，安全系统在隔离状态下很难判定其恶意性。多个子任务在时间顺序和语义依赖上组合后，却可能恢复出原始攻击目标。

DrAttack 揭示了“拆分”本身可以成为越狱攻击策略。该方法把恶意提示拆解为指令、结构、动词和名词等子提示，再借助良性示例诱导模型在上下文学习中隐式重构原始意图[6]。Safe in Isolation, Dangerous Together 将这一思想扩展到多智能体、多轮次场景，使用问题分解者、子问题回答者和答案组合者协同完成攻击。该研究指出，现有安全过滤器通常孤立评估提示词，缺少在多轮交互中推断组合意图的能力[7]。这一缺陷正是本文防御系统需要解决的问题。

从形式上看，可以将一个多轮攻击会话表示为查询序列：

$$Q=(Q_1,Q_2,\ldots,Q_n)$$

其中 $Q_i$ 表示第 $i$ 轮用户输入。传统单轮检测方法通常学习一个判定函数：

$$D_{single}(Q_i)\rightarrow\{\mathrm{allow},\mathrm{refuse}\}$$

该函数只根据当前输入给出允许或拒绝决策。多轮拆解攻击要求防御系统学习一个依赖历史上下文的判定函数：

$$D_{context}(Q_i, Q_{<i})\rightarrow\{\mathrm{allow},\mathrm{refuse}\}$$

其中 $Q_{<i}$ 表示当前轮之前的历史查询。若系统在第 $k$ 轮检测到当前输入补全或推进了历史查询中的风险链并拒绝，则该攻击样本被视为防御成功。若 $n$ 轮全部输入后系统仍未拒绝，则说明攻击穿透。

多轮拆解攻击的风险不一定体现在显式恶意关键词上，而常体现在任务阶段的累积上。攻击者可能先询问目标系统、相关资源或一般背景，再询问实现条件、工具选择、规避措施和执行步骤。每一轮都可以被包装成研究、教育或防护语境，但其组合结果可能形成可执行攻击链。安全检测需要识别的不是某个词是否危险，而是当前查询是否使会话从目标识别推进到能力获取、执行准备、规避控制或目标实现。

多智能体系统会进一步强化这种风险。在多 Agent 架构中，规划器可能将复杂目标拆解为局部任务，检索器负责寻找信息，执行器调用工具，汇总器组合结果。若每个 Agent 只看到局部输入，攻击者就可以利用系统分工隐藏全局意图。Wu 等（2024）将智能体系统建模为组件图，说明对抗信息可以在组件间流动并被反思或搜索机制放大[2]。本文虽然不直接实现多 Agent 攻击环境，但评估方式模拟了攻击者将有害目标拆解为多轮 benign 子任务的过程，因此需要会话级上下文建模。

## 2.3 知识图谱构建与 EDC 方法

知识图谱是一种以实体和关系为基础的结构化知识表示形式，通常可表示为：

$$G=(V,E)$$

其中 $V$ 表示节点集合，$E$ 表示边集合。对于自然语言中的语义事实，常使用三元组表示：

$$t=(s,r,o)$$

其中 $s$ 为主语实体，$r$ 为关系，$o$ 为宾语实体。若将多轮查询写入图谱，系统可以把每一轮输入表示为查询节点，把抽取出的对象表示为实体节点，把实体之间的语义联系表示为关系边。与普通长文本上下文相比，图谱能够显式保存“哪个查询提到了哪个实体”“哪个实体与哪个实体存在何种关系”“当前查询与历史查询通过什么关系连接”等信息。

知识图谱用于本文任务时有两个作用。其一，它是上下文记忆结构。多轮会话中的查询、实体和关系被持续写入同一 session 图谱，系统可以在任意轮次检索当前查询周围的历史证据。其二，它是推理证据结构。若当前查询与历史查询之间存在目标、能力、规避和执行等阶段性关系，图路径可以为拒绝决策提供可解释依据。相比把所有历史对话拼接成一段长文本，图谱方式可以减少无关上下文干扰，并把安全判断建立在可追踪的节点和边上。

自然语言输入到知识图谱之间存在语义抽取和关系规范化问题。用户查询通常不是结构化数据，同一关系可能有多种表达方式。例如“绕过限制”“规避检测”“避开安全策略”在文本表面不同，但在安全语义上可能都属于规避控制。若系统直接把原始关系写入图中，后续推理会面对大量分散、同义或近义的关系标签。Kumar 等（2020）使用预训练语言模型学习实体感知关系，说明预训练模型能够改善实体关系抽取[13]。Zhang 和 Soh（2024）提出的 EDC 方法则更适合开放域场景，因为它不要求所有关系在抽取前已经固定[14]。

EDC 是 Extract、Define、Canonicalize 三个阶段的缩写。Extract 阶段使用大语言模型从文本中开放式抽取候选三元组，不强制关系必须来自预设 schema。Define 阶段为抽取出的关系生成自然语言定义，使关系不只是一个短标签，而具有可比较的语义描述。Canonicalize 阶段将关系定义与目标 schema 中的关系定义进行相似度计算，并结合上下文判断是否可以映射为规范关系。若没有合适目标关系，也可以保留自定义关系或进行自规范化。

本文借鉴 EDC 的原因在于，多轮安全查询中的风险关系往往表达含蓄，固定关键词无法覆盖全部形式。通过为关系生成定义并执行向量检索，系统可以把不同表述归并到统一关系类型，使图推理层能够稳定识别风险阶段。Co-Guard 的语义解析模块在抽取三元组后，会对关系进行定义、候选 schema 检索和规范化处理。该流程为后续 relation-first 搜索提供了更稳定的关系集合，也使系统能够在规则后端和大模型后端之间保持一致的数据结构。

知识图谱构建还需要同时处理隔离与共享两种上下文边界。在场景一基线评估中，每条攻击样本绑定独立 `session_id`，所有查询节点、实体节点和关系边都记录所属会话，测试结束后清理该 session，以避免跨样本污染。在更接近真实交互的场景三中，系统进一步将 `context_id` 与 `session_id` 分离，使多条隐藏攻击链能够在同一长期 `user_context` 中交错推进，而外层 session 仍可轮换。运行时检索优先依据 `context_id`，评测器保留 `hidden_goal_id` 仅用于离线统计。该设计使图谱既能支持严格隔离的基线实验，也能支持长期上下文下的混合流评估。

国内知识图谱研究为这一设计提供了直接参考。漆桂林等（2017）总结了知识图谱的基本概念、关键技术与应用场景[47]。刘峤等（2016）从信息抽取、知识融合和知识加工三层梳理了图谱构建流程[48]。马忠贵等（2020）进一步将知识抽取、融合、推理和质量评估视为图谱全生命周期的关键环节[49]。封皓君等（2021）则指出，路径式和混合式推理在知识补全与问答任务中具有较好的解释性优势[50]。这些工作共同支持本文将多轮查询写入上下文图，并在局部子图上执行可追溯推理。

## 2.4 图推理与 Think-on-Graph 思路

图推理强调在结构化图谱中寻找与问题相关的节点、关系和路径，并基于这些路径完成判断。与直接依赖大语言模型内部知识相比，图推理具有更强的证据约束。Pan 等（2024）指出，知识图谱能够为大语言模型提供外部知识、事实约束和可解释路径[11]。Sun 等（2024）提出 Think-on-Graph，将大语言模型作为图上智能体，在知识图谱中交互式探索实体和关系，并通过搜索得到候选推理路径[16]。该思想说明，大语言模型不必一次性处理全部图谱，而可以围绕当前问题逐步检索相关子图。

RoG 方法从另一个角度说明了关系路径的重要性。Luo 等（2024）提出规划、检索、推理框架，先由模型生成关系路径规划，再沿知识图谱检索具体路径，随后将路径证据提供给大语言模型完成推理[17]。这种方式避免了把知识图谱只当作事实文本库，而是把图中的关系序列作为推理骨架。G-Retriever 则面向文本图问答，将节点和边编码为向量，通过 Top-K 检索找到相关散点，再用奖品收集斯坦纳树构建紧凑子图，并把文本化子图与图向量软提示共同输入大语言模型[18]。该方法表明，在大规模文本图中，直接展开全部图结构并不可行，检索和子图压缩是图上问答的关键步骤。

本文的安全检测任务与普通图问答不同。普通图问答关注从图中找到答案，本文关注当前查询是否与历史查询共同形成风险链。二者都需要从大图中选择相关证据，但本文更重视关系类型对风险阶段的表达。对于多轮拆解攻击，实体相似性只能说明两个查询讨论了相近对象，却不能说明会话是否正在从目标识别走向执行准备。关系类型能够表达当前动作在攻击链中的位置，例如 identifies_target、acquires_capability、plans_evasion、bypasses_guardrail 和 exfiltrates_secret 等关系比普通实体匹配更接近安全判定需求。

基于这一特点，本文采用 relation-first 多跳搜索策略。该策略以当前查询为中心，检索同一 session 中的上下文子图，并优先扩展风险关系、阶段关系和跨轮桥接关系。搜索得到的路径可以表示为：

$$P=(v_0,r_1,v_1,r_2,\ldots,r_m,v_m)$$

其中 $v_0$ 通常与当前查询相关，$r_j$ 表示路径上的第 $j$ 条关系。系统根据关系类型、实体风险、是否连接历史查询、是否触及危险主题和是否补全阶段链路计算路径分数。风险关系会提高路径分数，教育、解释、防护和合规语义会形成反证或降低风险。该设计使系统既能识别潜在攻击链，也能避免把所有安全讨论误判为恶意请求。

上下文组装评分器是对路径搜索的补充。多轮拆解攻击中的关键证据有时不会集中在一条短路径上，而分散在多轮查询和多个实体之间。该机制将当前查询与近期可见历史映射到任务阶段集合，比较当前轮是否新增了历史中缺失的阶段，是否与已有危险主题或内容锚点形成上下文对齐，是否使会话从准备阶段推进到目标实现阶段。若当前查询补全了风险链路，系统会提高链路分数，并将相关理由写入最终判定结果；若缺少主题或锚点对齐，则链路分数会被压低。该机制使 Co-Guard 能够处理长链路、弱连接和多阶段累积风险，同时降低共享上下文下的跨链污染。

从方法关系上看，本文并未直接复制 ToG、RoG 或 G-Retriever 的算法，而是吸收其共同思想：知识图谱提供结构化证据，大语言模型或规则推理模块围绕相关子图进行判断，最终结果应当能够追溯到节点、关系和路径。Co-Guard 将这一思想迁移到多轮子任务拆解攻击检测中，把图推理的目标从“回答问题”改为“识别风险链是否形成”。因此，本章介绍的相关理论为后续系统设计奠定了基础。第三章将说明 Co-Guard 如何把语义解析、上下文图谱、图推理与关键实现组织为完整防御流程。
